# 自反馈机制追踪范围评估报告

> 生成时间: 2026-03-24
> 状态: 草稿

## 一、概述

自反馈机制（Feedback Loop）是注意力智能增强系统的核心组件，通过追踪高注意力股票的价格变化，形成"观察 → 信号 → 反馈 → 学习"的闭环。

### 核心组件

| 组件 | 职责 |
|------|------|
| `AttentionTracker` | 跟踪注意力信号，记录入场价格和时间 |
| `PriceMonitor` | 按频率调度获取实时价格，计算PerformanceMetrics |
| `FeedbackLoop` | 接收价格反馈，触发 Bandit 在线学习 |
| `BanditUpdater` | Contextual Bandit 算法更新注意力策略权重 |
| `FrequencyScheduler` | 根据注意力级别动态调整更新频率 |

## 二、追踪范围精确评估

### 2.1 触发条件

**信号来源**: `AttentionStrategyBase.emit_signal()`

```
注意力策略扫描高注意力股票
       ↓
计算价格/成交量动量
       ↓
满足阈值条件 (combined_threshold >= 0.7)
       ↓
emit_signal(buy/sell)
       ↓
AttentionTracker.track_attention()
       ↓
注册到 FrequencyScheduler + PriceMonitor
```

**具体触发阈值**:

| 参数 | 值 | 说明 |
|------|-----|------|
| `price_threshold` | 3% | 价格突破阈值 |
| `volume_threshold` | 2.0x | 成交量放大倍数 |
| `combined_threshold` | 0.7 | 综合得分阈值 |
| `min_symbol_weight` | 2.0 | 最低个股权重 |
| `min_global_attention` | 0.4 | 全局注意力门槛 |

### 2.2 数量计算模型

#### 扫描范围

```python
# WeightPoolView.get_high_attention_symbols()
threshold = 2.0  # min_symbol_weight
n = 100          # 最大返回100只
```

**扫描范围**: ~100 只股票/轮（权重 >= 2.0）

#### 实际发出信号数量

```python
# base.py - AttentionStrategyBase
cooldown_period = 60.0s    # 单只股票冷却期
max_positions = 10-20     # 策略最大持仓数
```

| 限制因素 | 参数 | 影响 |
|----------|------|------|
| 冷却期 | 60-180s | 同只股票冷却期内不重复发信号 |
| 持仓上限 | 10-20 | 同时最多持有10-20只 |
| 置信度门槛 | 0.5 | prediction_score < 0.5 跳过 |
| 观察期 | 3600s | 1小时后自动关闭跟踪 |

**估算每轮信号数**: 5-15 个buy信号

#### 同时被跟踪数量

```
每轮新信号: 5-15 个
平均观察期: 30分钟-1小时
冷却期重入: 10-20%
同时在观察: 20-40 只股票
```

### 2.3 频率分布

根据 [FrequencyScheduler](./scheduling/frequency_scheduler.py) 的三级频率体系：

| 注意力级别 | 权重范围 | 占比估算 | 更新频率 | 同时跟踪数 |
|-----------|----------|----------|----------|------------|
| **HIGH** | >= 3.0 | 15-25% | 每1秒 | 5-10 只 |
| **MEDIUM** | 2.0-3.0 | 50-60% | 每10秒 | 15-25 只 |
| **LOW** | < 2.0 | 15-25% | 每60秒 | 5-10 只 |

**总更新负载**:
- HIGH: 5-10 只 × 1次/秒 = 5-10 次/秒
- MEDIUM: 20 只 × 1次/10秒 = 2 次/秒
- LOW: 8 只 × 1次/60秒 = 0.13 次/秒

**峰值总更新频率**: ~7-12 次/秒

## 三、对比分析

### 3.1 与 BanditVirtualPortfolio 的区别

| 指标 | AttentionTracker (自反馈) | BanditVirtualPortfolio |
|------|---------------------------|----------------------|
| 触发条件 | 策略发出 buy/sell 信号 | 需要实际成交 |
| 样本数量 | **高** (~20-40只同时跟踪) | **低** (取决于实际成交) |
| 学习速度 | **快** (观察开始即可学习) | **慢** (需等待平仓) |
| 反馈及时性 | 实时 (最高1秒间隔) | 延迟 (平仓后才知道结果) |
| 噪音风险 | 中 (可能跟踪假突破) | 低 (成交信号更可靠) |

### 3.2 样本量对比

**AttentionTracker (改进后)**:
- 每轮扫描: ~100 只候选
- 同时跟踪: ~25-40 只
- 每小时新增样本: ~40-80 个观察记录
- 每日样本: ~1000-2000 个

**BanditVirtualPortfolio (原有)**:
- 取决于实际成交率
- 估计每日成交: 10-50 笔
- 每日新增样本: 10-50 个

**结论**: 自反馈机制样本量是原来的 **20-100 倍**

## 五、结论与建议

### 5.1 当前状态评估

| 维度 | 评估 |
|------|------|
| 追踪范围 | 合理 (~25-40只同时跟踪) |
| 频率调度 | 已对接 (1s/10s/60s三级) |
| 样本产生速度 | 良好 (每小时40-80个样本) |
| 计算负载 | 可控 (峰值~12次更新/秒) |

### 5.2 潜在优化点

1. **观察期可动态调整**: 当前固定1小时，可根据波动率调整
2. **冷却期优化**: 180s 可能偏长，可缩短至60-120s
3. **多策略协同**: 目前主要依赖 MomentumSurgeTracker，可增加其他策略信号

### 5.3 风险提示

- 高频假突破可能导致噪音样本增加
- 样本质量依赖于动量策略的准确率
- 建议配合 BanditVirtualPortfolio 的真实成交反馈一起使用
