# 注意力系统 Web UI 配置说明

## 概述

已成功为注意力系统创建了 Web UI 页面，可以通过浏览器实时查看注意力状态和策略执行情况。

## 访问方式

启动 naja 后，访问：
```
http://localhost:8080/attentionadmin
```

或者在导航菜单中点击 **👁️ 注意力** 标签页

## 页面功能

### 1. 全局注意力卡片
- 显示当前全局注意力值 (0-1)
- 根据注意力水平显示不同颜色：
  - 🔴 红色 (≥0.7): 极高 - 市场极度活跃
  - 🟠 橙色 (≥0.5): 高 - 市场活跃
  - 🟡 黄色 (≥0.3): 中 - 市场正常
  - 🟢 绿色 (<0.3): 低 - 市场平静

### 2. 系统状态
- 运行状态：运行中/已停止
- 处理快照数
- 平均延迟

### 3. 策略概览
- 总策略数
- 活跃策略数
- 总信号数

### 4. 频率分布
- 🔴 高频股票数量及占比
- 🟡 中频股票数量及占比
- 🟢 低频股票数量及占比

### 5. 策略状态
显示每个策略的详细信息：
- 策略名称和状态（启用/禁用）
- 执行次数
- 生成信号数
- 跳过次数
- 优先级

### 6. 双引擎状态
- 🌊 River Engine：处理数、异常数、活跃股票数
- 🔥 PyTorch Engine：推理数、队列大小、处理中数量
- 双引擎触发次数

### 7. 最近信号
显示最近生成的交易信号：
- 🚀 买入信号
- 💨 卖出信号
- 👀 观察信号

## 控制按钮

- **🔄 刷新**：手动刷新页面数据
- **⏸️ 暂停刷新**：暂停/启用自动刷新

## 自动刷新

页面默认每 2 秒自动刷新一次，可以通过按钮控制。

## 技术实现

### 创建的文件

1. **`deva/naja/attention/__init__.py`**
   - 模块初始化文件

2. **`deva/naja/attention/ui.py`**
   - UI 渲染逻辑
   - 数据获取和格式化
   - 自动刷新控制

### 修改的文件

1. **`deva/naja/web_ui.py`**
   - 添加 `attentionadmin()` handler
   - 添加路由 `/attentionadmin`

2. **`deva/naja/common/ui_theme.py`**
   - 添加导航菜单项 "👁️ 注意力"

## 数据流

```
浏览器访问 /attentionadmin
    ↓
render_attention_admin() 渲染页面
    ↓
获取注意力系统报告
    ↓
获取策略统计
    ↓
渲染各个组件
    ↓
启动自动刷新循环
```

## 使用说明

1. 启动 naja：
   ```bash
   python -m deva.naja
   ```

2. 打开浏览器访问：
   ```
   http://localhost:8080/attentionadmin
   ```

3. 查看实时数据：
   - 观察全局注意力变化
   - 查看策略执行状态
   - 监控生成的信号

4. 控制刷新：
   - 点击 "暂停刷新" 停止自动更新
   - 点击 "刷新" 手动更新数据

## 注意事项

1. 页面数据每 2 秒自动刷新
2. 如果注意力系统未启动，页面会显示空状态
3. 信号列表只显示最近 20 个信号
4. 需要确保 `realtime_tick_5s` 数据源正在运行才能看到数据变化
